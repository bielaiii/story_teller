# 内容管理说明

小说内容使用 Markdown 持久化，但日常操作全部在网页中完成。请运行 `./run.sh`，不要直接编辑本目录中的源文件。

## 剧情

在“剧情”页面可以：

- 新建、修改和删除文章；
- 插入新章节，或修改已有文章的阅读顺序；
- 编辑标题、摘要、状态、颜色、标签、关键剧情和高潮标记；
- 手动补充出场人物与关联设定；
- 同步编辑 Markdown 正文和预览；
- 从“检查 → 内容回收站”预览或恢复 7 天内删除的文章。

剧情的 `id` 是稳定引用，创建后不会因为插入、移动或删除其他章节而变化。`sequence` 只负责阅读顺序，页面会自动维护连续编号。

## 时间线

在“时间线 → 编辑时间线”中管理：

- 新建、改名、排序和删除剧情线；
- 调整颜色、左右方向、分支点和汇合点；
- 将文章拖到剧情线或未编排区域；
- 删除剧情线时转移已有节点。

时间线只管理横向归属，纵向顺序始终来自文章阅读顺序。`timeline.md` 和文章的 `lanes` 由页面自动保存。

## 人物与关系

在“人物”页面可以新建人物，并在人物详情中修改完整档案、调整收纳状态、安全改名或删除。

人物档案支持简介、别名、头像路径、分组、标识、叙事定位、阵营、主线影响程度、图谱可见性和自定义档案字段。

在人物详情中可以编辑或删除已有关系；在“检查 → 新增人物关系”中创建关系。人物姓名修改会先显示受影响的正文和文件，确认后同步更新引用。

## 设定与灵感碎片

“设定”页面支持新建、编辑和删除组织、势力、地点、物品、事件背景与规则，并可维护别名、标签、区域、相关人物和补充剧情。

“碎片”页面支持新建、编辑、删除，以及将碎片转成正式剧情。转为剧情成功后，原碎片进入 7 天回收站。

## 作品、篇章与图谱

在“剧情 → 作品与篇章”中修改作品名称、顶部名称以及篇章名称、数量和顺序。仍有文章的篇章不能直接删除。

在人物图谱中拖动节点后，打开“布局设置”保存人物位置；这里也能调整节点间距、关系长度、外围延伸和自动布局力度。`graph-layout.md` 由页面维护。

## 检查与恢复

“检查”页面提供：

- 内容回收站：剧情、人物、关系、设定和碎片统一保留 7 天；
- 安全重命名：预览人物或设定名称影响后再应用，并支持撤销；
- 安全修复：整理文章顺序、规范人物与关系文件名并移除明确失效的引用；
- 配置检查：报告重复 ID、失效引用、歧义称呼和布局问题。

`content-index.json` 由本地服务自动扫描生成，`manifest.md` 只保存作品和篇章信息，不需要维护文件列表。
