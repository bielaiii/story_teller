# Demo 内容包

`story.db` 是本内容包的唯一可写数据源。Markdown、`project.snapshot.json` 和 `recovery.snapshot.json` 均由应用生成；日常修改请在网页中完成。

## 页面能力

- 剧情：新建、编辑、删除、篇章归属和阅读顺序；稳定 ID 不随插入或移动改变。
- 时间线：独立维护故事时间与剧情线归属；删除剧情线时必须显式选择接收线。
- 人物：完整档案、KV 事实、补充设定、互斥定位、安全重命名、人物关系和图谱入口。
- 设定与碎片：新增、编辑、删除、标签筛选和沉浸写作。
- 图谱：人物锚点、环绕、距离约束、视觉分组、平移和缩放。
- 检查：确定性诊断、分类回收站、正文预览、恢复和七天内操作撤销。

所有正文编辑器支持 Markdown 预览、双向同步滚动、目录跳转、沉浸模式、格式快捷键，以及 `@` 人物 / `/` 设定智能引用。保存和筛选只更新本地状态与增量数据，不刷新整个页面。

## 导出

- Markdown 文件用于阅读和 Git diff，不参与运行时读写；
- `project.snapshot.json` 用于静态只读部署；
- `recovery.snapshot.json` 用于在数据库丢失时完整重建实体、引用、顺序、回收站和操作历史；
- `timeline.md`、`graph-layout.md`、`manifest.md` 和 `content-index.json` 由应用维护，不需要手工编辑。
